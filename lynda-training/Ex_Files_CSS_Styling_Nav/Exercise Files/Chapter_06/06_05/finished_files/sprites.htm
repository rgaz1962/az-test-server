CSS Sprites

CSS Sprites allow us to reduce the overhead that icons and images typically add to sites. While they are not right for every project or image, they do add a valuable option for adding icons and other images to our site navigation. Sprites work by assembling multiple images into a single image. Images are spaced so that you can easily reference them in your CSS and exported to the desired image format as a single file.

To use sprites, you add the sprite image to the desired element by using the background-image property. Padding is added to the element to allow the image to display without the text overlapping, and then the background-image is positioned so that the appropriate image displays. For rollovers or other elements, the same image is used with the appropriate background-image positioning to reveal the desired image.

When using sprites, you should take care to make sure you have enough space around the individual images within the sprite to account for any padding or element height and width without displaying other images within the sprite. It’s also important to space images evenly, so that the values required to position each of the images is easier to arrive at. It’s also worth noting that the http savings they give you can be offset by the size of the sprite. If the sprite image grows too large, it could actually add more overhead than using separate images.
